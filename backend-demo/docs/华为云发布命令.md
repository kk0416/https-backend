# 华为云发布命令

## 1. 文档目的

1. 给当前 `gaussian-backend-demo` 提供一套可以直接参考执行的华为云发布命令。
2. 尽量与参考项目的发布方式保持一致。
3. 把需要你最终确认或替换的地方集中列出来。

## 2. 当前已收口的生产参数

### 2.1 镜像仓库

当前默认使用：

`swr.cn-north-4.myhuaweicloud.com/seer_develop/sep-gaussian-backend-demo`

### 2.2 Ingress 公网地址

当前默认使用：

`https://sep-gaussian-backend.cloud-data-dev.seer-group.com/`

### 2.3 共享 ELB

当前默认复用参考项目里已经验证可用的共享 ELB：

`32bb8857-86a4-475b-b80a-f650ab200a8a`

### 2.4 Image Pull Secret

当前默认使用：

`default-secret`

### 2.5 Ingress TLS Secret

当前默认使用：

`sep-gaussian-backend-ingress-tls`

### 2.6 StorageClass

当前默认使用：

`csi-disk-topology`

原因：

1. 当前项目是单副本 `SQLite`
2. 更适合块存储场景
3. `WaitForFirstConsumer` 对 CCE 调度更稳

## 3. 发布前建议先确认

1. 域名 `sep-gaussian-backend.cloud-data-dev.seer-group.com` 是否已经解析到共享 ELB
2. `sep-gaussian-backend-ingress-tls` 是否已经在目标命名空间创建
3. 命名空间中是否已经存在 `default-secret`
4. 集群中是否存在 `csi-disk-topology`

## 4. 推荐命名空间

下面示例统一使用：

`roboshop`

如果你要部署到其他命名空间，把命令中的 `roboshop` 一并替换掉。

## 5. 镜像构建与推送

### 5.1 构建镜像

```powershell
$tag = "0.1.0"

docker build -t sep-gaussian-backend-demo:$tag .
```

### 5.2 打标签

```powershell
$tag = "0.1.0"

docker tag sep-gaussian-backend-demo:$tag swr.cn-north-4.myhuaweicloud.com/seer_develop/sep-gaussian-backend-demo:$tag
```

### 5.3 推送

```powershell
$tag = "0.1.0"

docker push swr.cn-north-4.myhuaweicloud.com/seer_develop/sep-gaussian-backend-demo:$tag
```

## 6. 创建 Ingress TLS Secret

如果命名空间里还没有 `sep-gaussian-backend-ingress-tls`，可以先创建：

```powershell
kubectl create secret tls sep-gaussian-backend-ingress-tls `
  --cert=.\tls.crt `
  --key=.\tls.key `
  -n roboshop
```

## 7. 部署命令

### 7.1 使用 `values-production.yaml`

```powershell
helm upgrade --install gaussian-backend-v1 .\helm `
  -n roboshop `
  --create-namespace `
  -f .\helm\values-production.yaml
```

### 7.2 当前最终版命令已经显式指定 StorageClass

因为当前集群没有默认 `StorageClass`，所以建议直接使用：

```powershell
helm upgrade --install gaussian-backend-v1 .\helm `
  -n roboshop `
  --create-namespace `
  -f .\helm\values-production.yaml
```

### 7.3 如果要切换镜像标签

```powershell
$tag = "0.1.1"

helm upgrade --install gaussian-backend-v1 .\helm `
  -n roboshop `
  --create-namespace `
  -f .\helm\values-production.yaml `
  --set image.tag=$tag
```

2. 命名空间 `roboshop` 中是否已经存在 `default-secret`

## 11. 4.4 是什么意思

这里说的不是浏览器访问域名时的 Ingress 证书。

这里说的是：

1. Pod 内部这个 Nest 服务自己监听 `8443` 时使用的 `server.crt` / `server.key`
2. 对应 chart 里的字段是 `tls.existingSecret`

当前有两种做法：

1. 不配置 `tls.existingSecret`
   1. 容器启动时自动生成一套自签名后端证书
   2. 对当前第一版最省事
2. 配置 `tls.existingSecret`
   1. 由 Kubernetes Secret 挂正式 `server.crt` / `server.key`
   2. 更适合正式环境

对你这个第一版 SQLite 快速上线项目，我的建议是：

1. 先不启用后端证书 Secret
2. 继续让容器自动生成后端证书
3. 重点先保证 Ingress 前端证书是正确的

也就是：

1. 浏览器访问域名看到的证书，优先保证 `sep-gaussian-backend-ingress-tls`
2. Pod 内部后端证书，第一版先不额外折腾

## 12. 当前最终发布假设

当前这份命令和配置默认按下面假设执行：

1. 命名空间：`roboshop`
2. 域名：`sep-gaussian-backend.cloud-data-dev.seer-group.com`
3. StorageClass：`csi-disk-topology`
4. Ingress TLS Secret：`sep-gaussian-backend-ingress-tls`
5. Image Pull Secret：`default-secret`
6. 后端服务证书：不单独挂 Secret，继续让容器自动生成
