# 后端 Demo 使用说明

## 1. 项目位置

1. 项目根目录：`D:\docker-images\gaussian\backend-demo`
2. 数据库文件：`D:\docker-images\gaussian\backend-demo\data\roboshop.db`
3. 本地开发证书文件：`D:\docker-images\gaussian\backend-demo\certs\dev-cert.pfx`
4. 本地 `.env` 示例端口：`34430`

## 2. 当前技术栈

1. `NestJS`
2. `Fastify`
3. `Prisma 7`
4. `SQLite`

## 5. 如何启动服务

### 5.1 进入项目目录

```powershell
cd D:\docker-images\gaussian\backend-demo
```

### 5.2 可选：先恢复演示数据

```powershell
npm run db:seed
```

### 5.3 启动服务

```powershell
npm run start:dev
```

### 5.4 启动后验证

在浏览器中打开：

```text
https://127.0.0.1:34430/api/v1/health
```

如果返回 `code = 0` 且 `status = running`，说明服务已经启动成功。

也可以用兼容 Docker / Helm 探针的地址验证：

```text
https://127.0.0.1:34430/health
```

第一次访问时如果浏览器提示证书不受信任，这是正常现象。
原因是当前项目使用的是本地开发自签名证书，而不是正式 CA 证书。

## 6. 如何停止服务

### 6.1 如果你是在当前终端直接执行 `npm run start:dev`

直接按：

```powershell
Ctrl + C
```

### 6.2 如果服务是在后台启动的

可以执行：

```powershell
Stop-Process -Id <PID>
```

例如：

```powershell
Stop-Process -Id 17320
```

### 6.3 后台启动时相关文件

1. PID 文件：`docs/server.pid`
2. 标准输出日志：`docs/server.stdout.log`
3. 错误输出日志：`docs/server.stderr.log`

## 9. 启动配置说明

### 9.1 当前优先读取的环境变量

1. `SERVER_HOST`
2. `SERVER_PORT`
3. `SSL_CERT_FILE`
4. `SSL_KEY_FILE`

### 9.2 当前仍保留的兼容字段

1. `HTTPS_PORT`
2. `HTTPS_PFX_PATH`
3. `HTTPS_PFX_PASSPHRASE`

### 9.3 当前 HTTPS 证书读取优先级

1. 如果同时提供 `SSL_CERT_FILE` 和 `SSL_KEY_FILE`，优先按参考项目方式启动
2. 如果没有提供 `crt/key`，则退回本地 `pfx` 证书方式
3. 如果两种证书都不存在，服务启动会直接报错

## 10. 推荐验证顺序

1. 先执行 `npm run db:seed`
2. 再执行 `npm run start:dev`
3. 先打开 `GET /health`
4. 再打开 `GET /api/v1/health`
5. 再看 `GET /api/v1/dashboard/data-summary`
6. 再看 `GET /api/v1/data-assets`
7. 再看 `GET /api/v1/data-assets/tree`
8. 再看 `GET /api/v1/data-assets/graph`
9. 最后再测 `POST /api/v1/data-assets/:id/generate-point-cloud`

## 11. 当前演示数据说明

当前 seed 会生成：

1. `1` 个现场

3. `5` 条数据资产

1. `RAW -> POINT_CLOUD`
