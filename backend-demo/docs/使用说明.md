# 后端 Demo 使用说明

## 1. 项目位置

1. 项目根目录：`D:\docker-images\gaussian\backend-demo`
2. 数据库文件：`D:\docker-images\gaussian\backend-demo\data\roboshop.db`

## 2. 当前技术栈

1. `NestJS`
2. `Fastify`
3. `Prisma 7`
4. `SQLite`

## 3. 常用命令

### 3.1 安装依赖

```bash
npm install
```

### 3.2 开发模式启动

```bash
npm run start:dev
```

### 3.3 构建

```bash
npm run build
```

### 3.4 重新写入演示数据

```bash
npm run db:seed
```

### 4.1 推荐组合

1. `Qt Creator`：用于 `Qt/C++`
2. `VS Code`：用于当前 `TypeScript/NestJS` 后端

## 5. 如何启动服务

### 5.1 进入项目目录

```powershell
cd D:\docker-images\gaussian\backend-demo
```

### 5.2 可选：先恢复演示数据

```powershell
npm run db:seed
```

### 5.3 启动服务

```powershell
npm run start:dev
```

### 5.4 启动后验证

在浏览器中打开：

```text
http://127.0.0.1:3000/api/v1/health
```

如果返回 `code = 0` 且 `status = running`，说明服务已经启动成功。

## 6. 如何停止服务

### 6.1 如果你是在当前终端直接执行 `npm run start:dev`

直接按：

```powershell
Ctrl + C
```

### 6.2 如果服务是在后台启动的

可以执行：

```powershell
Stop-Process -Id <PID>
```

例如：

```powershell
Stop-Process -Id 17320
```

### 6.3 后台启动时相关文件

1. PID 文件：`docs/server.pid`
2. 标准输出日志：`docs/server.stdout.log`
3. 错误输出日志：`docs/server.stderr.log`

## 7. 服务地址

1. 本地地址：`http://127.0.0.1:3000`
2. 全局前缀：`/api/v1`

## 8. 已实现接口

### 8.1 健康检查

```text
GET /api/v1/health
```

### 8.2 数据总览

```text
GET /api/v1/dashboard/data-summary
```

### 8.3 数据列表

```text
GET /api/v1/data-assets?page=1&pageSize=20
```

支持参数：

1. `siteId`
2. `sceneId`
3. `dataType`
4. `status`
5. `page`
6. `pageSize`

### 8.4 树状图

```text
GET /api/v1/data-assets/tree
```

### 8.5 关系图

```text
GET /api/v1/data-assets/graph
```

### 8.6 任务列表

```text
GET /api/v1/tasks?page=1&pageSize=20
```

### 8.7 生成点云任务

```text
POST /api/v1/data-assets/:id/generate-point-cloud
```

## 9. 推荐验证顺序

1. 先执行 `npm run db:seed`
2. 再执行 `npm run start:dev`
3. 打开 `GET /api/v1/health`
4. 再看 `GET /api/v1/dashboard/data-summary`
5. 再看 `GET /api/v1/data-assets`
6. 再看 `GET /api/v1/data-assets/tree`
7. 再看 `GET /api/v1/data-assets/graph`
8. 最后再测 `POST /api/v1/data-assets/:id/generate-point-cloud`

## 10. 当前演示数据说明

当前 seed 会生成：

1. `1` 个现场
2. `1` 个场景
3. `5` 条数据资产
4. `4` 条任务
5. `4` 条操作日志

当前链路为：

1. `RAW -> POINT_CLOUD`
2. `POINT_CLOUD -> GAUSSIAN`
3. `POINT_CLOUD -> MAP_2D`
4. `POINT_CLOUD -> MAP_3D`

2. `auth` 模块只是预留骨架

4. 如果测试了 `generate-point-cloud`，建议再执行一次 `npm run db:seed` 恢复演示初始状态
